# Feature Specs

One file per feature. Filename in kebab-case matching the feature or Jira epic (e.g. `user-auth.md`, `payment-flow.md`).

See `.claude/skills/prd-spec.md` for the full spec template and enrichment process.

## Status legend
- **Draft** — being written
- **In Review** — awaiting architect/team review
- **Approved** — ready for implementation
- **In Progress** — feature-builder is working on it
- **Complete** — shipped
