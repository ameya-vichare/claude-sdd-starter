# Claude Code Starter Kit

A platform-agnostic Claude Code setup for engineering teams. Provides an agentic development workflow with spec-driven feature development, multi-agent orchestration, and an interactive setup wizard that tailors everything to your project.

## Quick Start

### Copy into an existing project

```bash
git clone https://github.com/ameya-vichare/claude-sdd-starter
cp -rn claude-sdd-starter/.claude claude-sdd-starter/docs claude-sdd-starter/tasks \
       claude-sdd-starter/PROJECT.md \
       /path/to/your-project/
```

If `.claude` already exists in your project, copy the commands manually:

```bash
cp claude-sdd-starter/.claude/commands/claude-setup.md \
   claude-sdd-starter/.claude/commands/new-feature.md \
   /path/to/your-project/.claude/commands/
```

### Option 1 — Interactive Setup (recommended)

Run the setup wizard:

```
/claude-setup
```

The wizard will:
1. Scan your codebase and detect platform/language/framework
2. Ask clarifying questions about your project and goals
3. Install Context7 and any relevant MCP tools
4. Generate a project-specific PRD (`docs/PRD.md`)
5. Generate a tailored `CLAUDE.md` (~200 lines, zero placeholder text)
6. Generate skills, agents, and commands customized for your stack

### Option 2 — Manual Setup

1. Fill in `CLAUDE.md` with your project's architecture invariants
2. Fill in `docs/PRD.md` with your product requirements
3. Customize skills and agents for your language/framework
4. Start building with `/new-feature`

## What's included

```
.claude/
├── commands/
│   ├── claude-setup.md   ← /claude-setup — interactive project setup wizard
│   └── new-feature.md    ← /new-feature — start any new feature
├── agents/
│   ├── orchestrator.md   ← coordinates agents, tracks tasks, owns integration
│   ├── architect.md      ← evaluates structure, writes ADRs, produces diagrams
│   ├── feature-builder.md ← writes production code from approved plan
│   ├── code-reviewer.md  ← reviews PRs for correctness + architecture
│   └── test-writer.md    ← writes unit and integration tests
└── skills/
    ├── prd-spec.md       ← spec writing and enrichment guide
    └── pr-checklist.md   ← pre-PR verification checklist

docs/
├── PRD.md                ← product requirements
├── specs/                ← one spec per feature (auto-generated by /new-feature)
└── decisions/            ← architecture decision records (written by architect agent)

tasks/
├── lessons.md            ← top 10 lessons (Claude reads at session start)
└── todo.md               ← current plan in progress

PROJECT.md                ← live module registry (updated by orchestrator)
CLAUDE.md                 ← project conventions + architecture invariants
```

## The Two Flows

### Route A — Full Feature
For anything non-trivial: new screen, new module, new service boundary, 3+ files, new API endpoint.
1. Developer provides lightweight spec (Goal / Constraints / Done / Not specifying)
2. Claude enriches spec → `docs/specs/<feature>.md`
3. `architect` agent produces HLD/LLD/data flow diagrams
4. Developer approves plan
5. `orchestrator` records task breakdown in `tasks/<feature>.md`
6. `feature-builder` implements layer by layer (build-verified at each step)
7. `test-writer` writes tests
8. `code-reviewer` reviews PR
9. `refactor-agent` (optional) improves code quality

### Route B — Simple Task
For small changes: ≤2 files, additive, obvious path, <30 min.
1. Claude implements directly
2. Build + test + lint verification
3. Commit

## Agent Model Allocation

| Agent | Model | Role |
|-------|-------|------|
| orchestrator | claude-sonnet-4-6 | Coordinates agents, never writes code |
| architect | claude-opus-4-6 | Evaluates structure, writes ADRs |
| [lang]-researcher | claude-sonnet-4-6 | Platform research via Context7 |
| feature-builder | claude-sonnet-4-6 | Writes production code |
| code-reviewer | claude-sonnet-4-6 | Reviews PRs |
| test-writer | claude-sonnet-4-6 | Writes tests |
| ci-fixer | claude-sonnet-4-6 | Fixes build/CI failures |
| refactor-agent | claude-sonnet-4-6 | Post-review code improvement |

## Supported Platforms

The setup wizard detects and supports:
- **Mobile**: iOS (Swift), Android (Kotlin), Kotlin Multiplatform, React Native / Expo, Flutter (Dart)
- **Frontend**: React, Next.js, Vue, Angular, Svelte/SvelteKit (TypeScript or JavaScript)
- **Backend**: Node.js, Python (Django/Flask/FastAPI), Go, Rust, Java (Spring), Scala, Ruby (Rails), PHP (Laravel/Symfony), Elixir (Phoenix), C# (.NET)
- **Systems**: C/C++, Zig, Rust
- **Full-stack**: Monorepo or multi-repo setups

## Prerequisites

- [Claude Code CLI](https://docs.anthropic.com/en/docs/claude-code) installed
- Node.js (for Context7 installation via `npx ctx7 setup`)
